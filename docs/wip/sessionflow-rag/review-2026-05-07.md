## Feature Usefulness

V2 is meaningfully more useful than v1 for the target problem. Issue #41 reports that session indexing preserves conversational text but drops high-value tool-mediated reasoning, especially PAL prompts and tool-heavy exploration context. The v2 design solves that with selective enrichment rather than capture-all indexing.

Artifact comparison:

| Metric | v1 | v2 |
| --- | ---: | ---: |
| Chunks | 31 | 34 |
| Actual PAL prompt blocks | 0 | 6 |
| `[Read: ...]` lines | 0 | 41 |
| `[Write: ...]` lines | 0 | 5 |
| `[Edit: ...]` lines | 0 | 30 |
| `[Agent: ...]` lines | 0 | 2 |
| Raw `mcp__pal__` in titles | 7 | 0 |
| `mcp__capy__*` in metadata/titles | 22 | 0 |
| `Bash` in metadata/titles | 27 | 0 |

The v2 dump contains actual PAL prompt content for the whitelist/pattern-based extraction discussion, TotalAssistantChars design review, and selective enrichment tradeoffs. It also replaces noisy title strings like `Tools: Bash, Read, mcp__capy__capy_search, mcp__pal__chat` with cleaner `PAL: chat | Tools: Read, Agent` titles.

Known limitations remain as designed: PAL tool results are not captured, and non-PAL tool-only turns can still collapse into adjacent narrative turns.

## Implementation Correctness

The implementation mostly matches the design:

- `contentBlock.Input` is parsed.
- Exact PAL tools are promoted into assistant text with delimiter blocks.
- Read/Write/Edit/Grep/Agent are enriched into `ToolMeta`.
- Unknown tools, Bash, and capy self-calls are skipped by registry policy.
- `ToolMeta` flows through parsing, sub-agent parsing, transcript rendering, and sanitization.
- PAL chunk titles are grouped under `PAL:` instead of raw MCP names.
- Full tests and race tests pass.

Two implementation gaps remain:

1. Empty or malformed Promote-tier tool input still adds the tool name to `ToolNames`, so chunk titles can advertise `PAL: chat` even when no PAL content was promoted.
2. The temporary `cmd/dump-session/` helper remains in the branch even though task 6.5 says it must be deleted before shipping.

## Findings

### P2 - Promote tools with empty extraction still affect chunk titles

Location: `internal/session/parse.go:328`

Profile: go / code-style, generic correctness  
Triggered by: `.go` parser changes

`extractAssistantBlocks` appends `b.Name` to `toolNames` for `ActionPromote` even when `ext.Extract` returns an empty string. The design says Promote extraction failure should silently drop the block. Current behavior can create a `PAL: chat` title boost for malformed or empty PAL input even though there is no PAL delimiter block in the transcript.

Confidence: 90%. Verified directly in `parse.go:328-332`; tests cover the extractor returning empty strings but do not cover the parser-level `ToolNames` side effect.

Suggested fix: move `toolNames = append(toolNames, b.Name)` inside the `if extracted != ""` block and add a parser test for malformed/empty PAL input.

### P2 - Temporary dump-session command is still shipped in the branch

Location: `cmd/dump-session/main.go:1` and `docs/wip/sessionflow-rag/tasks-v2.md:83`

Profile: go / removal-plan  
Triggered by: new `cmd/dump-session/main.go`

Task 6.5 explicitly says `cmd/dump-session/` is temporary and not part of the shipped CLI, but the branch still adds it as a buildable command. It also hardcodes `# Raw v1 Session Dump`, which already mislabels the v2 artifact.

Confidence: 95%. Verified the task item and the command source.

Suggested fix: delete `cmd/dump-session/` before merge. If this utility should survive, move it to an explicitly internal/script location, rename the heading, and document it as supported.
