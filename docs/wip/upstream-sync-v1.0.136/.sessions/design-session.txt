  Section 1: Phrase Frequency Reranker (P1, search quality)

  The rerank function in search.go:240 already computes title-match boost and
  proximity boost (minSpan). The upstream adds a third additive signal: phrase 
  frequency boost ‚ÄĒ counting how many times consecutive query terms appear as
  adjacent pairs within 30 characters of each other.

  The implementation adds a countAdjacentPairs(positionLists [][]int, terms 
  []string, gap int) int helper. For each consecutive pair of terms (terms[i], 
  terms[i+1]), it sweeps left positions against right positions: for each left
  position p, it looks for the nearest right position within [p + len(terms[i]),
   p + len(terms[i]) + gap]. Each right position is consumed at most once
  (greedy left-to-right matching), so "foo foo bar" counts 1 pair, not 2.

  The boost formula is 0.5 * min(1.0, adjacentPairs/4) ‚ÄĒ saturating at 4 hits to
   prevent keyword-stuffed documents from dominating. The cap of 0.5 sits below
  max proximity (~1.0) and in the title-boost range (0.3-0.6), so it breaks ties
   without overwhelming the other signals.

  This integrates into the existing multi-term proximity block in rerank() (line
   267-316). After computing minSpan, we compute adjacentPairs using the same
  posLists already built for the span calculation, then add phraseBoost to the
  total boost alongside titleBoost + proximityBoost. No interaction with
  capy-specific features (synonym expansion, entity boosting, per-source
  diversification) ‚ÄĒ those run at different stages of the pipeline.

‚óŹ Section 4: Security Hardening (4 changes)

  4a. SSRF guard improvements (tool_fetch.go + new internal/server/ssrf.go): Two
   additions:

  Scheme validation: Before DNS resolution, parse the URL and reject non-HTTP/S
  schemes. Block file://, gopher://, javascript:, data:, and any scheme that
  isn't http or https. This runs first since it's the cheapest check.

  DNS rebinding defense via custom Transport: The current validateFetchURL
  resolves DNS once, but Go's http.Client performs its own resolution for the
  actual TCP connection ‚ÄĒ an attacker with a low-TTL DNS record can serve a
  public IP for the pre-flight check and then 169.254.169.254 for the connect.
  Fix: create a custom http.Transport with a DialContext that resolves DNS via
  net.Resolver, classifies every returned IP using the existing isLocalOrPrivate
   logic, and only dials if all IPs pass. This collapses the TOCTOU window to
  zero ‚ÄĒ DNS resolution and IP classification happen at connect time, not in a
  separate pre-flight step. Extract IP classification into classifyIP(ip net.IP)
   error in a new ssrf.go file for testability. The existing validateFetchURL
  becomes a scheme-only check; IP classification moves entirely to the
  Transport.

  4b. TOCTOU fix for stale auto-refresh: When refreshStaleSources() (from ¬ß2)
  re-reads a file, it must re-check the deny policy before reading. The file's
  deny status may have changed since original indexing. Implementation: accept
  an optional DenyChecker func(path string) bool on ContentStore (set by the
  server at init). In refreshStaleSources, before os.ReadFile(filePath), call
  s.denyChecker(filePath) ‚ÄĒ skip the source if denied. Fail-closed: if the
  checker returns an error, skip the re-read rather than proceeding.

  4c. Path traversal bypass (security/eval.go): EvaluateFilePath currently
  matches deny globs against only the raw input string. A relative path like
  ../../.ssh/id_rsa bypasses an absolute glob like /home/user/.ssh/**. Fix:
  accept an optional projectRoot string parameter. When provided, resolve the
  input path to absolute (filepath.Join(projectRoot, filePath) then
  filepath.Clean), then match deny globs against both the raw input AND the
  resolved path. Either matching triggers denial. Update checkFilePathDenyPolicy
   in security_check.go to pass s.projectDir as the project root.

  4d. Executor env deny list expansion (executor/env.go): Add .NET/C# runtime
  hijack environment variables to deniedEnvVars. These include CORECLR profiler
  attach vectors (CORECLR_PROFILER, CORECLR_PROFILER_PATH and arch variants,
  CORECLR_ENABLE_PROFILING), diagnostic IPC (DOTNET_DiagnosticPorts),

  COMPlus_* back-compat prefix. For COMPlus_*, use a prefix check in
  BuildSafeEnv alongside the existing BASH_FUNC_ prefix check rather than
  enumerating every variant.
